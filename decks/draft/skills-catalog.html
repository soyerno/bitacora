MODO 04 · Timing · por qué importa hoy

Por qué ahora

Hay tensión entre tener todo y usar lo que sirve.

01 · El problema

Discoverability colapsa con 190 skills

El listado a los agentes pesa ~5k tokens por turno. El skill correcto se diluye entre los 180 que no aplican. Resultado: o se invoca al que matchea por keyword, o no se invoca ninguno.

02 · La oportunidad

El core de MODO ya está identificado

  • modo-deck · decks branded en minutos
  • guardia + modo-code-review-checklist · PR review
  • sdd-* + modo-tech-architect · specs y arquitectura

Framing: no necesitamos quitar skills, necesitamos un mapa que diga cuáles son el core diario y cuáles on-demand. Sin ese mapa, el equipo subutiliza el set entero.

MODO 09 · Scope · core diario vs on-demand

Scope propuesto

El core diario son ~15 skills. El resto, on-demand.

Core diario · invocar siempre que aplique

  • modo-design-system + modo-deck · cualquier UI o presentación MODO.
  • sdd-* · obligatorio en /comercios, recomendado en todo cambio no trivial.
  • guardia + modo-code-review-checklist · PR review.
  • next-best-practices + react-doctor · trabajo en frontend.
  • modo-tech-architect · cuando arranca proyecto desde PRD.
  • modo-promos / consultar-promos-modo · cualquier dato de promos.
  • ruflo memory_store / search · invocación implícita, ya está en flow.

On-demand · solo cuando aplica

  • Migraciones puntuales (nextjs-12-to-15, tailwind-3-to-4, etc).
  • Generación de assets visuales (banners, iconos, mockups).
  • Skills "de gusto" (taste, impeccable, minimalist, brutalist): pick 1 según el output.
  • Vercel plugin: solo en repos hosteados en Vercel (promos-hub-site).
  • MWDD: a evaluar si reemplaza o complementa SDD.
  • POCs / agentes (agente-cx-whatsapp, simular-compra-agentica).
