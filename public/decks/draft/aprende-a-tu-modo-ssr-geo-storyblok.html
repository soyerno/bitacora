04 · Por qué juntas

arquitectura

Storyblok + GEO + SSR
se potencian, no se solapan.

A

Storyblok sin SSR

El editor puede publicar pero el crawler no lo ve. El cambio en Storyblok llega al user via JS post-paint, no al HTML inicial. GEO trap sigue presente.

B

GEO sin SSR + Storyblok

Inline LDs en index.html cubren home, no per-curso. Cada curso nuevo del editor pide deploy. No escala.

C

SSR sin Storyblok

Tenés HTML estático per curso pero el contenido sigue hardcodeado en TS. Cualquier cambio editorial vuelve a ser deploy.

D

Las tres juntas

Storyblok como source. SSG genera HTML per slug con LD scoped al curso. Editor publica, ISR/build cron re-genera. Crawler indexa cada curso.
