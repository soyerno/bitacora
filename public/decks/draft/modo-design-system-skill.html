03 · DiagnósticoEl estado hoy

El estado hoy

5 repos, 4 stacks distintos, 1 brand que va a la deriva.

Cada dev frontend tiene que reaprender los tokens MODO cada vez que entra a un repo nuevo. La lib @playsistemico/modo-ui-lib-web tiene SCSS variables. modo-landing tiene 3 styling systems conviviendo. promos-hub-site está en Next 15 + Tailwind v4. Las fuentes están en _document.js, los colores en tailwind.config.js, los CSS vars en globals.css, y los SCSS tokens en la lib externa.

Síntomas observados

Hex hardcoded en componentes~60 ocurrencias
Páginas sin JSON-LD breadcrumb12 de 28 routes
Variantes de fonts en usoInter, system-ui, Arial leak
Container max-width inconsistente1200, 1260, 1440, 92vw
Letter-spacing display ausente~40% de los H1

Costo

Tiempo onboarding dev nuevo a tokens~3 días
PRs con feedback "no es el color MODO"recurrente
Tiempo crear página nueva from scratch~1 día
Tiempo review de SEO + JSON-LD~2h por PR
Drift visual entre repos hermanosvisible

→ commit: feat(SEGUROS-001): scaffold page

11 · Learning loopIterativo · cada run mejora

Iterativo · el skill aprende del uso real

3 mecanismos de mejora continua.

M1

runs.jsonl

Cada generación deja entry: timestamp, repo, slug, tipo, tokens detectados fuera de DS, correcciones del user post-gen. Grep-able, append-only.

M2

learnings.md

Si el user corrige el mismo patrón ≥3 veces, se abre entry: detectado, síntoma, fix aplicado, status. Cada cierre dispara update de template y bump version.

M3

ds-snapshots/

Run trimestral de extract-tokens.sh graba sha256 de tailwind.config + globals.css. Diff detecta drift del DS upstream y dispara revisión del skill.

# Ejemplo de loop cerrado (hipotético)
2026-06-10  user genera /seguros, agrega class="text-emerald-600" post-gen
2026-06-15  user genera /promos-banco-galicia, agrega class="text-emerald-600" post-gen
2026-06-22  user genera /promociones-cyber, agrega class="text-emerald-600" post-gen
            → learnings.md crea A007 "text-emerald-* sustituye sdk-system-success"
            → review: ¿agregar nuevo token al DS? ¿bloquear text-emerald-*?
            → resolución: agregar al token sdk-colors-success-emphasis
            → bump v1.0.0 → v1.0.1
            → template update
            → próxima generación ya lo aplica
