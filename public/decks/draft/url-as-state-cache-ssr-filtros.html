04 · Por qué juntas

timing

SEO, cache y share son la misma palanca:
URL diferenciada por estado.

A

Si hacés URL-state sin cache

Servís SEO indexable pero el origen come N veces más requests. Cada combo de Googlebot pega al BFF. Costo de servidor sube sin proporción.

B

Si hacés cache sin URL-state

No hay cache key estable. El CDN o cachea mal (user A ve content de B) o el equipo apaga cache entera. Es lo que pasa hoy.

C

Si hacés SEO sin SSR

El catálogo viene client-side filtered. Crawler descarga 500 items y los filtra en JS que no corre. Default state es lo único que se indexa.

D

Las tres juntas

URL-state + SSR subset + cache headers per cache-class. Una sola migración, tres ganancias. Después solo se mantiene.

Decision tree

  • Eje taxonómico, 1 valor → segmento /promos/categoria/[slug]
  • Combinable → query ?categoria=...&banco=...
  • Search → query ?q=... con noindex
  • Sort, page → query con noindex,follow

Sitemap

  • Top 5 categorías + top 5 bancos como entries
  • Long-tail combinatoria queda fuera
  • Generado dinámicamente desde catálogo MODO

  • SearchAction en el WebSite del home

  • Google Rich Results test por estado representativo
  • Comparación de numberOfItems vs itemListElement.length

07 · Pilar 03 · Cache key = URL

pilar 03

1 → 25

Entries calientes,
cardinality controlada.

URL es la cache key. Sin ella, todos los filtros colapsan en una entry. Con whitelist de top combos, cada estado popular se cachea independiente.

Policy 4 clases

ClaseAplica aHeader
longDefault + top singles-maxage=600 swr=3600
mediumSingle filter no-tops-maxage=300 swr=1800
shortCombo 2 filtross-maxage=60 swr=300
noneSearch, sort, long-tailprivate no-store

Invalidación

  • Tag-based con revalidateTag('promos') en Next 15
  • Webhook Storyblok / CMS lo dispara con ISR_REVALIDATE_TOKEN
  • Cero enumeración de URLs en código de invalidación

09 · Scope

qué entra y qué no

Filtros públicos sí.
Estado de UI privada no.

ENTRA

  • /promos modo-landing · categoría, banco, día, búsqueda
  • Root promos.modo.com.ar · mismos ejes
  • Paginación de listados
  • Sort cuando aplica (noindex)
  • JSON-LD scoped + sitemap top combos
  • Cache headers per cache-class

NO ENTRA

  • Modal open/close, dropdowns, accordions, tooltips
  • Drawer mobile, animation step, skeleton state
  • Form draft (sigue en localStorage)
  • Auth state (cookies / session)
  • /pagar checkout (flujo privado)
  • Combinatoria long-tail indexable (queda noindex)

/comercios (en QA, todavía no live) entra en una fase posterior cuando se promueva a prod, junto con el programa Next 16.
