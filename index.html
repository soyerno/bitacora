Herramientas

Hecho con Claude Code · skills utilizados

Cada deck se genera con la skill modo-deck como orquestadora y un conjunto de skills auxiliares que garantizan calidad de diseno, prose discipline y brand fidelity. La version final se versiona en este repo. Las decisiones de comms y voz estan ancladas en el skill modo-deck.

modo-deck — orquestadora
Genera presentaciones HTML branded MODO con voz rioplatense, comms discipline y storytelling Pyramid Principle. Punto de entrada para todo deck en este repo.
modo-design-system
SDK MODO para generar paginas Next.js/Astro one-shot con el design system MODO (Quicksand + Red Hat Display, verde #008859, brand tokens canonicos). Auto-invocable para paginas en cualquier repo del workspace MODO.
impeccable github →
Skill de Phil Bakaus para diseno frontend production-grade. Aplica anti-patterns checks: sin gradientes, sin nested cards, color restrained, sin em dashes.
taste-skill github →
Skill de Leonxlnx con anti-slop frontend rules. Evita over-design y mantiene la interfaz limpia y funcional.
ui-ux-pro-max github →
Skill de NextLevelBuilder con 161 paletas, 57 font pairings, 50+ estilos de UI. Referencia de composicion visual y tipografia.
writing-clearly-and-concisely
Strunk para prose: una claim por oracion, sin buzzwords, sin verbos de relleno. Garantiza que cada slide diga exactamente lo que tiene que decir, sin padding.
storyblok — MCP server
Acceso al CMS de MODO. Permite leer y escribir contenido del sitio directamente desde el agente durante la generacion del deck cuando se necesita contexto real.
chrome-devtools — MCP server
Para capturar screenshots reales de paginas en localhost o produccion. Garantiza que los decks con capturas de pantalla usen imagenes reales, no mocks.

Regla operativa: cada deck se versiona en github.com/SoyErnoModo/modo-decks. Las decisiones de comms y voz se anclan en el skill modo-deck. Para generar un deck nuevo: /modo-deck en Claude Code, contexto del deck, y el agente orquesta el resto.
